# MailGPT

An intelligent email automation platform that integrates with AI models to automatically process, analyze, and respond to emails. Part of TheUseless.AI platform.

## Features

- **Automatic Email Processing**: Receives emails via AWS SES/SNS webhooks
- **AI-Powered Responses**: Generates contextual replies using LLM models
- **Email Thread Context**: Maintains conversation history for intelligent continuation
- **Multiple Recipient Support**: Handles To, CC, and BCC scenarios
- **Customizable AI Personas**: Configure characters, storylines, and game rules
- **Marketplace**: Share and discover AI personas and interaction rules

## Architecture

```
AWS SES → SNS → Django Webhook → Mail Fetcher Queue
                                        ↓
                              GPT Reply Fetcher Queue
                                        ↓
                                Mail Sender Queue → SMTP
```

### Django Applications

| App | Description |
|-----|-------------|
| `core` | User authentication, mail agent configuration |
| `xmail_engine` | Email data models and conversation management |
| `studio` | AI model configuration, character/storyline/game rule management |
| `market` | Marketplace for Characters, Storylines, and Game Rules |

### Queue Workers

| Queue | Purpose |
|-------|---------|
| `mail_fetcher` | Receives and parses incoming emails |
| `gpt_reply_fetcher` | Generates AI replies using LLM |
| `mail_sender` | Sends AI-generated replies via SMTP |
| `system_mail_sender` | Sends system notifications |

## Quick Start

### Prerequisites

- Python 3.11+
- PostgreSQL 15+
- Redis 7.0+

```bash
# MacOS
brew upgrade
brew install redis postgresql
redis-server

# Debian/Ubuntu
sudo apt-get install redis-server postgresql

# Create virtual environment
python3 -m virtualenv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Set environment variables
cp .env.example .env
```

### Database Setup

```bash
# Apply migrations
python server/manage.py migrate --settings=server.settings.dev_common

# Create admin user
python server/manage.py createsuperuser --settings=server.settings.dev_common

# Load AI model fixtures
python server/manage.py loaddata fixtures/studio.aimodel.json --settings=server.settings.dev_common
```

### Start Server

```bash
# API Server (port 8000)
python server/manage.py runserver 127.0.0.1:8000 --settings=server.settings.dev_api

# Admin Server (port 9000)
python server/manage.py runserver 127.0.0.1:9000 --settings=server.settings.dev_admin
```

### Start Redis Queue Workers

```bash
python server/manage.py rqworker mail_fetcher --settings=server.settings.dev_common
python server/manage.py rqworker gpt_reply_fetcher --settings=server.settings.dev_common
python server/manage.py rqworker mail_sender --settings=server.settings.dev_common
python server/manage.py rqworker system_mail_sender --settings=server.settings.dev_common
```

## Tech Stack

- **Framework**: Django 5.1, django-ninja
- **Database**: PostgreSQL
- **Queue**: Redis Queue (RQ)
- **AI/LLM**: LangChain
- **Email**: IMAPClient, mail-parser
- **Auth**: JWT (python-jose)
